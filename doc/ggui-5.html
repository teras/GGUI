5. Creating new datafiles

At last! From version 0.4.0 there is an internal option to graphically create a new datafile or edit an old one. There is no need to manually edit the text file anymore. Still, if you still want to see the documentation about the format of the data file you can have a look at the GGUI datafile format section.

Although it is not difficult to understand how to create a new file, I prefer to give step-by-step information so that anyone can easily create such a file. What to do:

5.1 Preperation stage

First you need to gather some general information.

  • Decide for which program you want to create a UI.
  • Learn which are its parameters. To do so run it without any parameter or with the "-h" or "--help" or "-help" parameter, or even something like "-?". Something should work and you will see a list of all possible parameters.
  • If these pages don't give enough information, try other sources. Type man {program-name}
  • Use the info pages
  • Read the possible documentation of this program (usually found under this directory "/usr/doc/.../target-program" - in my system is under "/usr/doc/packages/target-name" )
  • Make sure you noted all the possible or most needed switches and you understood the usage of them.

5.2 Managing parameters.

Think over each parameter :

  • Is it a switch (on & off)?
  • Does it have more than one possibilities?
  • What is the default state of this parameter (when it's ommited)?
  • Do you HAVE to use it?
  • Do you have to write extra text for this parameter?

Because you have to visually appear them on your screen as a wizard think about this too:

  • Can you group some parameters together?
  • How many groups can you make?
  • Do you need to show all the parameters or some of them are not so importand and can be completely ommited?
  • Some parameters are importand, but only for experienced users? You can use the interactive wizard possiblity to create an "Advanced option" section, so that it will not be neccesary for an unexperienced user to see all these parameters.
  • Which group is more importand?
  • Should some perameters been put before others?
  • Can the user be able to exit the wizard in the middle, and use the default values for the remaining ones?

5.3 Creating the data file.

It's time now to write the data file. First you have to decide what name to give to the new datafile. If you give a name for an already existing datafile, then GGUI will edit that file. If this filename can't be found, GGUI will create this new file. The command to use is ggui -create datafile, where datafile is your desired name. You should not provide the ".ggui" extension. The wizard will automatically append it for you. You don't even have to worry where to save this file - this is done automatically too. If you want information about the screen go to the description of the Creator screen section.

5.4 Transform the parameters of the target-application to GGUI objects

Now it is time to "fill the gaps". What you can do:

  • Start adding pages. For any group of parameters use a special wizard page.
  • For every page start adding objects, depending the type of parameter.
  • The more important parameters/objects and pages of course should be as close to the beginning as possible.
  • The advanced parameters (if any) should be at the end pages.
  • It's a good tradition to leave the last page of the wizard without any parameters, only displaying a text like "Click on finish to go on." so that the wizard will be more understandable and much easier to handle for interactive wizards.
  • If you want an image to display on the left you have to write the filename of the image for every page. All images should be saved under the GGUI_HOME/img directory.
  • Some objects can be displayed with various GUI types. Select the one which seems to be more apropriate.
  • Take attention with the spaces. Usually they are not importand, but sometimes they are.
  • If you want a page to automatically jump to another page and not to the next in the row, you can set interactivity to an info object for example and define the first page to jump to in a Page section to whatever you want to.

5.5 Test it

Type ggui datafile, where datafile is the name you gave when you created this datafile.The wizard should pop. Play a little bit with it. Is it functional? Can others understand this wizard? Is it pretty? What do you think of the results? If you are not satisfied try to correct the datafile (edit it again). Run the wizard once more. Remember what you choosed and click on the "Finish" button. Are the parameters as they supposed to be? Is there any error? Does the target-program run correctly? Are there any errors? Does it understand correctly all parameters? Play a little bit with the parameters.

5.6 Installation

No more installation is neccesary. Still if you want you can make even easier the usage of the ggui environment. E.g. if you want to automatically run the ggui wizard with the xterm data everytime you type xterm at your command prompt, you can add this line to your .bashrc file: 

alias xterm="ggui xterm"
